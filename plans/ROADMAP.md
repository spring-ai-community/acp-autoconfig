# Roadmap: ACP Spring Boot Autoconfiguration

## Overview

Implementation proceeds from infrastructure through client-side autoconfiguration, then agent-side, then testing support. This order lets each stage build on the previous — transports before clients, clients before agents, all before integration tests.

---

## Stage 1: Foundation

### Step 1.0: Design review
- [ ] Review VISION.md, DESIGN.md with stakeholder
- [ ] Confirm property prefix and naming conventions
- [ ] Confirm autoconfiguration class structure
- **Exit**: Design approved, no open blockers

### Step 1.1: Project scaffolding
- [ ] Parent POM with Spring Boot 4.0.5 BOM and ACP SDK dependency management
- [ ] `acp-spring-boot-autoconfigure` module POM
- [ ] `acp-spring-boot-starter` module POM

- [ ] `.gitignore`

## Stage 2: Client Autoconfiguration

### Step 2.1: Client transport autoconfiguration
- [ ] `AcpClientTransportAutoConfiguration` class
- [ ] WebSocket transport bean (conditional on URI property)
- [ ] Stdio transport bean (conditional on command property)
- [ ] Auto-detection when `type` is not set
- [ ] `@ConditionalOnMissingBean` for user override
- [ ] Tests with `ApplicationContextRunner`
- **Exit**: Transport beans created/skipped correctly based on properties

- [ ] `AcpClientAutoConfiguration` class
- [ ] `AcpAsyncClient` bean with configured timeout and capabilities
- [ ] `AcpSyncClient` bean wrapping async client
- [ ] `@ConditionalOnMissingBean` for both
- [ ] `DisposableBean` for graceful shutdown
- [ ] Tests: client beans created when transport present, skipped when absent
- [ ] Tests: user-defined client bean overrides auto-configured one
- **Exit**: Client beans auto-configured and lifecycle-managed

### Step 2.3: Client integration test
- [ ] End-to-end test: auto-configured client talks to `MockAcpAgent` via `InMemoryTransportPair`

- **Exit**: Integration test passes

## Stage 3: Agent Autoconfiguration

### Step 3.1: Agent transport autoconfiguration
- [ ] `AcpAgentTransportAutoConfiguration` class
- [ ] Stdio transport bean (default)
- [ ] WebSocket transport bean (conditional on Jetty class + property)
- [ ] `@ConditionalOnMissingBean` for user override
- [ ] Tests with `ApplicationContextRunner`
- **Exit**: Agent transport beans created correctly

### Step 3.2: Agent annotation bootstrap
- [ ] `AcpAgentAutoConfiguration` class
- [ ] Discover `@AcpAgent`-annotated beans from application context
- [ ] Wire through `AcpAgentSupport` with transport and interceptors
- [ ] `SmartLifecycle` for start/stop
- [ ] Fail-fast for zero or multiple `@AcpAgent` beans
- [ ] Tests: agent starts on context refresh, stops on close
- [ ] Tests: interceptor beans auto-wired in order
- **Exit**: Annotation-driven agent fully lifecycle-managed

### Step 3.3: Agent integration test
- [ ] End-to-end test: `@AcpAgent` bean handles prompts from auto-configured client
- [ ] Verify full protocol round-trip through `InMemoryTransportPair`
- **Exit**: Integration test passes

## Conventions

- Each step ends with: run tests, create learnings if applicable, update ROADMAP checkboxes, commit
- Commit format: imperative subject, optional body
- Entry criteria for each stage: previous stage complete and tests passing
- Learnings go in `plans/learnings/step-N.M.md`
